## ⭐ OS评测系统设计（os-eval.html）

### 评测维度与指标

| 维度 | ①级 | ②级 | ③级 |
|------|------|------|------|
| **自动化水平** | 能单行补全 | 能跨文件重构 | 能自主完成完整feature |
| **逻辑推理深度** | 能回答简单逻辑题 | 能追踪多步因果链 | 能发现用户逻辑矛盾并提出替代方案 |
| **成本效率** | 单任务成本合理 | 批量任务有规模经济 | 提供了成本优化建议 |
| **人类和谐度认知** | 不冒犯用户 | 能理解用户情绪并调整语气 | 能主动推进协作，理解上下文中的未言明意图 |
| **专业适配度** | 能处理通用任务 | 在特定领域（科研/金融/法律）有显著优势 | 能根据任务类型自动切换工作模式 |
| **生态开放性** | 有API | 有插件/扩展生态 | 开源核心 + 社区贡献活跃 |

### 种子评测（CodeWhale示例）

| 维度 | 评分 | 理由 |
|------|------|------|
| 自动化 | ②跨文件重构 | 支持多文件patch和并行工具调用，但复杂feature仍需人工拆解 |
| 逻辑推理 | ③发现矛盾 | Constitution机制能在修改前验证一致性 |
| 成本效率 | ②批量经济 | 前缀缓存约90%成本折扣，长session有优势 |
| 人类和谐度 | ②情绪调整 | Calm personality响应得体，但缺少主动推进能力 |
| 专业适配 | ②领域优势 | 编程/工程领域突出，科研写作一般 |
| 生态开放 | ①API | 闭源，无插件生态，社区在GitHub但仓库不公开 |

### 用户评分系统
- **打分**：5星制（每维度独立打分）
- **标签投票**：每个评测对象有预设标签（如"适合科研""适合创业"），用户可投票
- **防刷**：注册7天以上+至少完成1次对抗或发散使用，方可评分
- **质量排序**：评价字数>50字优先展示，被点赞多的评价上浮

```
首页→场景卡片"决策验证"→交叉验证流：
  提出战略假设→多模型从不同立场分析→对比分歧地图→形成决策备忘录
  可选：OS评测帮选团队AI编程工具
  可选：工作流市场获取行业分析模板
```

## 🌐 GitHub开源 + 论文 + 网页版 三维联动

### GitHub：单仓三层结构

```
aicoper/
├── methodology/          ← ① 评测方法论（论文+数据集）
│   ├── paper/            # arXiv预印本(LaTeX+PDF)
│   └── benchmarks/       # 标准化评测数据集(JSONL)
├── web/                  ← ② 网页版源码
│   └── [所有HTML+CSS]
├── README.md             # 含Badge/Demo GIF/对比表/Quick Start
├── CONTRIBUTING.md       # 社区贡献指南
└── LICENSE               # Apache 2.0
```

### 学术论文：arXiv优先
- **平台**：arXiv（cs.AI / cs.HC）— 免费、快速（2-3天上线）、AI领域标准
- **选题建议**："Multi-Model Adversarial Evaluation: A Framework for Cross-Vendor AI Benchmarking and Collaborative Reasoning"
- **创新点**：多模型对抗评测方法论 + 发散模式认知价值 + 聚合器架构设计
- **时间线**：写作3-5天 → arXiv提交 → 2-3天上线 → 同步到HuggingFace Papers

### 网页版工具
- **技术**：GitHub Pages静态托管（免费、CDN、自动部署）
- **实现**：评测展示 + 官方链接跳转。不代理API，不嵌入iframe
- **部署**：GitHub Actions自动从 `web/` 目录部署

### Star增长冷启动
1. **arXiv论文上线当天** → HackerNews "Show HN" + Reddit r/MachineLearning
2. **论文被引用后** → ProductHunt上线
3. **国内**：知乎专栏 + v2ex + 即刻
4. **持续**：每新增一个模型评测 → Twitter线程 + 更新arXiv v2
