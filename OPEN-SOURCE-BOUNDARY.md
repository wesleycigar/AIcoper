# AIcoper 开源边界与治理模型

> 版本：V1.0  
> 日期：2026-06-17  
> 原则：最大化社区参与，保护核心商业可持续性

---

## 一、开源什么（Apache 2.0 / CC BY 4.0）

| 层级 | 内容 | 许可证 | 目的 |
|------|------|--------|------|
| **L1 评测方法论** | 对抗模式/发散模式的形式化定义、评分算法、OS评测维度 | CC BY 4.0 | 建立学术可信度，允许被引用和改进 |
| **L2 评测数据集** | 标准化评测数据（JSONL格式）、评测记录、场景模板 | CC BY 4.0 | 社区共建的核心资产——越多贡献，评测越准 |
| **L3 前端代码** | HTML/CSS/JS（全部页面）、设计系统 | Apache 2.0 | 允许任何人部署、二次开发、提交PR |
| **L4 论文** | arXiv预印本、方法论白皮书 | CC BY 4.0 | 学术传播，允许引用和衍生 |

## 二、不开源什么

| 内容 | 原因 |
|------|------|
| **API密钥/聚合引擎后端** | 商业敏感——多模型调用的路由逻辑、成本优化算法是核心运营壁垒 |
| **用户系统/认证** | 隐私和安全——涉及用户数据、评分记录、贡献档案 |
| **合作社返利逻辑** | 商业模式——两级返利的具体实现、即时激励分发算法 |
| **工作流市场内容** | 会员权益——资深用户沉淀的实战方法论属于付费内容 |
| **实时评测数据（生产环境）** | 运营资产——社区的实时评分和评价是平台的竞争壁垒 |

---

## 三、为什么这样分？

**开源方法论+数据** → 吸引研究者，建立学术信誉。LMSYS的成功证明：评测框架越开放，被引用越多，影响力越大。

**开源前端** → 降低参与门槛。任何人都能部署一个本地版本，提交UI改进，甚至fork一个定制版——这创造生态而非竞争。

**保留后端+商业逻辑** → 保证商业化可持续。评测方法论免费，但调用多模型的聚合服务、会员专属的深度功能、工作流市场——这些是付费用。

**核心原则**：开源"怎么评测"和"评测什么"，保留"怎么运营"和"怎么赚钱"。

## 四、社区贡献路径

```
外部研究者/开发者
    │
    ├─ 提交新模型评测数据 → PR到 benchmarks/
    ├─ 质疑现有评分 → 开 Issue，附带证据
    ├─ 改进前端 → PR到 web/
    ├─ 提出新评测维度 → 开 Issue 讨论 → 被采纳后加入方法论文档
    ├─ 翻译文档 → PR 多语言 README
    └─ 发表引用论文 → arXiv 引用 AIcoper 方法论

社区贡献者
    │
    ├─ 高质量贡献者 → 邀请成为「社区评审员」
    ├─ 持续贡献者 → 获得合作社贡献加成（更高折扣）
    └─ 核心贡献者 → 受邀加入方法论委员会
```
